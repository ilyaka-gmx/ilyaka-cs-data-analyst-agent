# Requirements Mapping

How each assignment requirement maps to our implementation.

## Mandatory Requirements


| Requirement                  | Implementation                                                                                                                             | Comments                                                                                                                                                                                                                                                                  |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Query Router**             | `src/router.py` — `classify_query()` using `ROUTER_MODEL` (Qwen3-30B) with structured JSON output                                          | Classifies into 4 types: `structured`, `unstructured`, `recommend`, `out_of_scope`. Uses a dedicated cheaper model, not the agent model. Tested with 17 cases in `tests/test_router.py`.                                                                                  |
| **Tools + Pydantic schemas** | `src/tools.py` — 11 tools, each with `@tool` decorator, Pydantic `BaseModel` input schema with `Field(description=...)`, and typed returns | 7 data tools + `summarize_responses` + 3 memory tools (`remember_fact`, `recall_profile`, `update_profile`) + `recall_past_sessions`. All schemas use `Field(ge=, le=)` for input bounds. 27 tests in `tests/test_tools.py`.                                              |
| **Multi-step reasoning**     | `src/agent.py` — manual ReAct loop (`agent_step` ↔ `tool_step`) inside a LangGraph `StateGraph`                                            | Agent calls multiple tools iteratively (e.g., `recall_profile` then `count_rows`). Loop detection prevents infinite cycles. DSML XML repair handles malformed tool calls. Tested via `test_multi_step_shows_tool_calls`.                                                  |
| **CLI + reasoning output**   | `main.py` — `--session`, `--user`, `--health` args; prints tool calls + results + per-query token usage                                    | `print_reasoning_steps()` shows each tool call and its result before the final answer. Token tracking reports input/output tokens per query.                                                                                                                              |
| **Max iterations fallback**  | `MAX_ITERATIONS=12` in `agent_step` + `recursion_limit=12` on graph                                                                        | When iteration limit is reached, agent returns a graceful "I've been working on this but couldn't complete" message. Both inner loop and outer graph limits enforced.                                                                                                     |
| **Conversation memory**      | `SqliteSaver` checkpointer in `checkpoints.db`                                                                                             | Full LangGraph conversation persistence. `--session` maps to `thread_id`. Follow-up queries ("show me 3 more") resolve correctly. Independent sessions verified by 5 dedicated tests in `tests/test_agent.py`.                                                            |
| **User profile**             | `src/memory.py` — [mem0](https://github.com/mem0ai/mem0) with Qdrant in-process vector store; tools: `remember_fact`, `recall_profile`, `update_profile` | Memory-as-tools pattern: agent autonomously decides when to save facts. Mem0 handles semantic deduplication and embedding-based recall. LLM-based fact extraction (Qwen3-30B) + Qwen3-Embedding-8B for vectorization. Profile survives restarts. 7 profile tests in `tests/test_agent.py`. |
| **MCP server**               | `src/mcp_server.py` — FastMCP v3 server exposing 7 data tools                                                                              | Each MCP tool delegates to the existing LangChain tool via `.invoke()`. Memory tools excluded (they require agent-level user context). 12 tests in `tests/test_mcp.py` using in-process `Client`.                                                                         |


## Bonus Requirements


| Requirement           | Implementation                                                                         | Comments                                                                                                                                                                                                                                                                                                                              |
| --------------------- | -------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Web UI**            | `frontend/index.html` — single-file SPA; `api_server.py` — FastAPI backend             | Custom HTML/CSS/JS frontend (no build step). SSE streaming with live reasoning steps. Tabbed layout: Chat, Admin (execution traces), Memory Insights (4-panel dashboard). Sidebar: user picker, conversation list with search, theme selector. Also: `streamlit_app.py` as legacy fallback. 19 API tests + 21 UI tests.               |
| **Query recommender** | `src/recommender.py` + `recall_past_sessions` tool + router `recommend` classification | Two paths: (1) API-driven suggestion chips via `/api/recommend` endpoint using `ROUTER_MODEL`; (2) Agent inline mode when router classifies `recommend` — agent calls `recall_past_sessions` + `recall_profile`, suggests queries without executing, waits for user confirmation. Past session context is opt-in via frontend toggle. |


## Features on top of Requirements


| Feature                                       | Implementation                                                                        | Comments                                                                                                                                                                                                                                                               |
| --------------------------------------------- | ------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Cross-model quality scoring (Judge)**       | `src/quality.py` — `JUDGE_MODEL` (Llama-3.3-70B) scores each response on 3 dimensions | Post-response scoring using a different model family than the agent. Evaluates `data_grounded`, `addresses_question`, `conciseness` (1–5 each). Judge card rendered beside each response in the frontend. Toggleable per-session. 14 tests in `tests/test_quality.py`. |
| **Dynamic tool exposure**                     | `get_tools_for_query_type()` in `src/tools.py`                                        | Router classification drives which tools are bound to the LLM on each call. Structured queries get data+memory tools; unstructured adds `summarize_responses`; recommend gets memory-only. Reduces per-call tool description tokens by ~40%.                           |
| **TOON output format**                        | `src/toon.py` — `to_toon()` formatter                                                 | Multi-record tool outputs use TOON (field names once as header, pipe-delimited rows). 30–60% fewer tokens than JSON for tabular results.                                                                                                                               |
| **Middleware harness**                        | `src/middleware.py` — `TokenTrackingMiddleware`, `ToolTimingMiddleware`               | Real `AgentMiddleware` subclasses with `wrap_model_call` and `wrap_tool_call` hooks. Token counts and tool durations tracked per query and per session. Message trimming (last 30) replaces LLM-based summarization.                                                   |
| **Session store + tracing**                   | `src/session_store.py` — `SessionStore`, `QueryTrace`                                 | Per-query execution traces: query type, tokens, duration, tool calls, memory ops. Powers Admin tab in the frontend and `/api/admin/stats`.                                                                                                                             |
| **Health checks**                             | `src/health.py` — `run_startup_checks()`, `run_diagnostics()`                         | Validates dataset, API connectivity (agent + router models), persistence. Available via `--health` CLI flag, `/api/health` endpoint, and Streamlit sidebar.                                                                                                            |
| **Automated test suite**                      | 8 test files, 138 test functions                                                      | Covers tools, router, agent integration, API endpoints, UI helpers, MCP server, quality scoring, and connectivity. Both fast (mocked) and slow (real LLM) tests with `@pytest.mark.slow`.                                                                              |
| **Model selection with documented rationale** | `src/config.py` — configurable via `.env`                                             | Three models (Agent: Qwen3-235B, Router: Qwen3-30B, Judge: Llama-3.3-70B). DeepSeek-V3.2 and Llama-3.3-70B rejected as agent models with documented reasons (DSML leak, infinite loops). All overridable via environment variables.                                    |
| **Zscaler/corporate proxy handling**          | `src/config.py` — `get_http_client()`                                                 | Auto-detects `combined_ca_bundle.pem` for corporate proxy. Zero-config for graders (no `.pem` → standard SSL). Required for the specific cases when Zscaler is used on dev machine                                                                                     |
| **Semantic memory (mem0)**                    | `src/memory.py` — mem0 with Qdrant in-process vector store                            | LLM-based fact extraction with semantic deduplication and embedding-based recall. Replaces JSON profile files. Qdrant runs in-process (local disk). Configurable via `MEM0_LLM_MODEL`, `EMBEDDING_MODEL`, `MEM0_STORAGE_PATH`.                                          |
| **Tool definition enrichment**                | `src/tools.py` — WHAT/WHY/WHEN docstrings                                              | Enriched tool docstrings guide the LLM on tool selection, usage patterns, and fallback strategies. Reduces tool selection confusion, especially for open-source models.                                                                                                 |
| **Query decomposition**                       | `src/agent.py` — `decompose_step` node; `src/prompts.py` — `DECOMPOSITION_PROMPT`     | Pre-execution planning: decomposes ambiguous queries into search strategies before the ReAct loop. Uses cheap router model. Toggleable via `ENABLE_DECOMPOSITION`.                                                                                                      |
| **Agent reflection**                          | `src/agent.py` — `reflect_step` node; `src/prompts.py` — `REFLECTION_PROMPT`          | Post-response self-check: detects "lazy" answers and nudges the agent to try harder. Single retry. Toggleable via `ENABLE_REFLECTION`.                                                                                                                                 |
| **Runtime model selector**                    | `frontend/js/admin.js` + `api_server.py` — `/api/models`, `/api/model`                | Browse available Nebius models with quality ratings, pros/cons, and per-token pricing. Switch agent model at runtime without restart. Model catalog in `src/model_catalog.py`.                                                                                           |
| **Frontend UX features**                      | `frontend/js/` — modular JS with shared `App` namespace                                | Chat search and filtering, tag management (auto + manual), feature toggles sidebar, status bar with model/dataset/token info, cache-busting for static assets.                                                                                                         |
